# Весточка — контекст проекта

**Весточка** — веб-сервис, который даёт пользоваться российским мессенджером **MAX** прямо из
**Telegram**, не устанавливая MAX на телефон. На сервере держится сессия MAX пользователя и связывается
с ботом в Telegram: входящие из MAX прилетают в Telegram, ответы уходят обратно в MAX. Технический «мост»
лежит в отдельном проекте (`../максим`); этот репозиторий — **сайт** (лендинг + блог + регистрация).

## Аудитория

Русскоязычные, у кого близкие в России пишут в MAX; часто уехавшие / живущие между странами; ценящие
приватность и не желающие ставить ещё одно приложение. Подробнее — `.claude/blog/HOUSE_STYLE.md`.

## Что умеет мост (для точности в текстах)

- В обе стороны: текст, ответы-цитаты, фото, голосовые, видео-кружки, файлы. Видно, кто пишет в группах.
- Уведомления о новых сообщениях приходят как обычные сообщения Telegram.
- Ограничения (честно): старая история не переносится; коды входа на Госуслуги MAX показывает только в
  своём приложении на телефоне (переслать нельзя); стикеры и реакции пока не передаются.
- Регистрация по телефону (SMS) или почте, без входа через Google/Apple (требование РФ).

## Стек сайта

Next.js (App Router) + React 19 + Manrope, деплой на Coolify (домен vestochka.uk), авто-деплой по git push.

## Блог

Файловый: посты — MDX в `src/content/blog/*.mdx` (frontmatter: title, slug, description, date, author, tags, draft).
Загрузчик — `src/lib/blog.ts`, страницы — `src/app/blog`. Добавить пост = положить .mdx в папку.

## Контент-пайплайн

`/blog-write [тема]` → агенты `blog-topic-picker → blog-researcher → blog-drafter → blog-critic` пишут
готовый пост в `src/content/blog/`. Правила: `.claude/blog/HOUSE_STYLE.md` (голос) + `.claude/blog/SEO.md` (поиск).

## Skill routing

При запросе на статью в блог — вызывать `/blog-write`.
